## 🏗️ How I Built This Portfolio

Most people think GitHub Pages is only for static HTML + CSS + JS files you upload manually. **Not this one.** This portfolio is a full **Vue 3 SPA** with a modern build pipeline, deployed automatically via **GitHub Actions** — just like Vercel or Netlify.

### Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Framework** | Vue 3 (Composition API, `<script setup>`) | Reactive UI with clean, modern syntax |
| **Router** | Vue Router 4 | SPA navigation without page reloads |
| **Build Tool** | Vite 6 | Lightning-fast HMR and optimized production builds |
| **Styling** | Tailwind CSS 3 | Utility-first CSS with custom design tokens |
| **Language** | JavaScript (ES Modules) | Native module system, no transpiler bloat |

### Animation & Interaction Stack

| Package | Version | What It Does |
|---------|---------|--------------|
| **GSAP** `^3.12.5` | Core animation engine | Timelines, tweens, sequenced entrances |
| **ScrollTrigger** (GSAP plugin) | Scroll-driven animations | Parallax, reveal-on-scroll, scrubbed effects |
| **@studio-freight/lenis** `^1.0.42` | Smooth scrolling | Inertia-based scroll synced to GSAP's ticker |
| **CSS** | Native | Mesh gradient background, glassmorphism, magnetic hover effects |

**Why no Framer Motion?** This is a **Vue** project. GSAP is framework-agnostic and pairs perfectly with Vue's lifecycle hooks (`onMounted`) for fine-grained control over ScrollTrigger instances.

### Design Philosophy (Awwwards-style)

- **Dark theme** `#1C1D20` — near-black with subtle warmth
- **Typography** — Inter font, massive scale contrast (clamp for fluid sizing)
- **Layout** — Dennis Snellenberg-inspired: massive name marquee, horizontal parallax, mouse-follow image previews
- **Micro-interactions** — Magnetic buttons with elastic snap-back, staggered scroll reveals, bouncy wave emoji
- **SPA feel** — Page transitions with fade + scale, no hard reloads

### Deployment Pipeline

```
Git push to main
        │
        ▼
┌─────────────────┐
│  GitHub Actions │  ← Ubuntu runner, Node 24
│  workflow runs  │
└─────────────────┘
        │
        ▼
   npm install   ← Fresh install (lockfile deleted for cross-platform safety)
        │
        ▼
   vite build    ← Bundles, tree-shakes, hashes assets, outputs to dist/
        │
        ▼
   Upload dist/  ← GitHub Pages artifact
        │
        ▼
   Live at        ← https://radityaich.github.io/
   radityaich.github.io
```

**This is CI/CD.** Every `git push` to `main` triggers the entire pipeline. No manual FTP. No drag-and-drop. No `index.html` copy-paste. Just code → commit → live.

### Workflow File

`.github/workflows/deploy.yml`

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: ["main"]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
      - run: |
          rm -rf node_modules dist package-lock.json
          npm install
      - run: npx vite build --emptyOutDir
      - uses: actions/upload-pages-artifact@v3
        with:
          path: './dist'

  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/deploy-pages@v4
```

### Key Configs

| File | Setting | Why |
|------|---------|-----|
| `vite.config.js` | `base: '/'` | Root-domain hosting (not `/repo-name/`) |
| `src/router/index.js` | `createWebHistory(import.meta.env.BASE_URL)` | Reads Vite's base path dynamically |
| `package.json` | `"type": "module"` | Native ES modules throughout |
